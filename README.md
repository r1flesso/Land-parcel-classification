# Land-parcel-classification
ML-проект на Python для классификации типов земель (на основе реальных данных) с использованием scikit-learn, imbalanced-learn (SMOTE, RandomOverSampler, RandomUnderSampler), optuna, MLflow. Сравниваются разные модели машинного обучения, а также подходы, решается проблема дисбаланса классов

## О проекте
В данном проекте я решил попробовать новые подходы машинного обучения и новые библиотеки для решения проблемы дисбаланса классов. Сначала я выбрал алгоритм машинного обучения - Random Forest. В прошлом своем проекте (https://github.com/r1flesso/Satellite-view-of-the-reservoir) я именно этот алгоритм использовал в качестве лучшей модели. Сейчас все то же самое. Основные преимущества Random Forest:
- Способность находить нелинейные, сложные зависимости
- Один из наиболее устойчивых к дисбалансу классов алгоритмов
- Интерпретируемость
Затем я обучил модели с разными комбинациями гиперпараметров, причем "подкручивал" их с помощью Optuna. Также я не забыл про новые технологии - библиотеку imblearn, в которой мне были интересны SMOTE, RandomOverSampler и RandomUnderSampler для добавления синтетических данных или удаления данных мажоритарного класса. Обучив все необходимые мне модели, я выбрал самую лучшую и прогнал ее еще раз с использованием Optuna.
Новым подходом, который я применил, было каскадное машинное обучение. Оно заключается в использовании нескольких моделей машинного обучения, каждая из которых дает либо финальный результат, либо результат для следующей модели, которая в свою очередь имеет такой же функционал. В моей структуре было 3 модели (A, B, C). Модель A распознавала, редкий этот класс или нет; модель B распознавала, какой это мажоритарный класс; модель C распознавала, какой это миноритарный класс. К сожалению, результаты получились очень плохие, но было очень полезно попробовать новый метод на практике, причем на реальных данных.

## Установка
1. Клонируйте мой репозиторий:
    ```bash
    git clone https://github.com/r1flesso/Land-parcel-classification
    ```
2. Создайте и активируйте виртуальное окружение:
   ```bash
   python -m venv venv
   source venv/bin/activate
   ```
3. Установите необходимые библиотеки:
   ```bash
   pip install -r requirements.txt
   ```

## Использование
1. Скачайте датасет для работы с ним и поместите в корень папки src (т.е. в саму папку):
   ```bash
   https://www.kaggle.com/datasets/timur15/types-of-territories
   ```
2. Прежде чем обучать модели, нужно сначала разбить датасет на разные обучающиеся выборки (тренировочную, валидачионную и тестовую). Для этого запустить файл split.py:
   ```bash
   python split.py
    ```
   С выборками ничего не надо делать
3. Обучите оставшиеся модели в папке src (названия всех файлов с моделями начинаются с random_forest)
4. Соберите все данные о результатах обучения моделей:
   ```bash
   python collect_hyperparameters.py
   ```
   А затем поместите таблицу в папки notebooks и src
5. Запустите лучшую модель:
   ```bash
   python rf_weights.py
   ```
6.1. Разбейте датасет на три разных датасета для трех разных моделей, запустите ноутбук datasets_for_cascade.ipynb
6.2. Запустите:
   ```bash
   python split_A.py
   ```
   ```bash
   python split_B.py
   ```
   ```bash
   python split_C.py
   ```
6.3. Запустите три файла для обучения моделей, которые нужны для финальной модели:
   ```bash
   python rf_smote_A.py
   ```
   ```bash
   python random_forest_B.py
   ```
   ```bash
   python random_forest_C.py
   ```
6.4. Запустите финальную модель:
   ```bash
   python final_model.py
   ```

## Структура проекта
- Папка cascade содержит файлы для реализации каскадного ансамбля моделей
- В папке notebooks представлены Jypyter notebooks, каждый из которых отвечает за свой этап экспериментов или служит для работы с датасетами
- Папка scripts имеет только один .py файл, который собирает метрики с моделей Random Forest
- В папке src содержатся файлы для обучения Random Forest с разными подходами (кастомный словарь весов, добавление в датасет синтетических данных...)
- Файл requirements.txt служит для управления зависимостями, а также содержит в себе необходимые библиотеки и их версии
